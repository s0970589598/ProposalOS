# 架構 — 明陽食品冷鏈監控

⚠️ **虛構案例 — 用於教學示範**

## 業務架構

```
冷凍倉（3 個）          配送車（12 台）
    ↓                       ↓
[溫度感測 + 告警]      [車載 IoT + GPS]
    ↓                       ↓
              中央雲端平台
                  ↓
    ┌─────────────────────────────┐
    ↓             ↓               ↓
員工後台      客戶查詢頁      司機 App
（廠務 / 品管） （QR 連結）    （配送員）
                  ↓
              ERP 整合
              （月報 + 履歷）
```

## 資料架構

```
即時資料（高頻）
├── 倉內溫度（每 1 min 採樣、每 5 min 上傳）
├── 車內溫度 + GPS（每 5 min）
└── 設備狀態（壓縮機、開關門）

批次資料（低頻）
├── 進料 / 加工 / 出貨批號（per 事件）
├── 客戶訂單（從 ERP 同步、每 1 hr）
├── 司機配送單（每天 09:00）
└── 月度報表（月底自動產出）

資料儲存
├── 即時資料 → 時序 DB（最近 30 天熱資料）
├── 歷史資料 → S3（≥ 6 個月，符合上市櫃稽核）
└── 批號履歷 → RDS（永久保存）

資料治理
├── 個資（員工、司機）→ 加密、最小存取
├── 客戶資料 → DPA 簽訂、只給授權客戶
└── 食品履歷 → WORM 不可竄改、留 7 年（食安法）
```

## 技術架構

```
雲端：AWS ap-northeast-1（東京）— 資料不離開亞洲

網路：
├── VPC（10.0.0.0/16）
├── Public Subnet（API Gateway、CloudFront）
├── Private Subnet（Lambda、RDS、Timestream）
└── 全部走 IAM Role、無 Long-lived credentials

資安：
├── WAF（前端 + API）
├── KMS（資料加密 at-rest）
├── TLS 1.3（傳輸加密）
├── Secrets Manager（API key、DB password）
├── CloudTrail（操作稽核）
└── GuardDuty（威脅偵測）

監控：
├── CloudWatch（系統指標、日誌、告警）
├── X-Ray（分散追蹤）
└── 自家 Dashboard（OEE、SLA、資料完整率）

部署：
├── GitHub Actions CI/CD
├── Terraform IaC
├── Blue-Green for 後端服務
└── DB migration with down() rollback
```

## 整合介面

| 對方系統 | 方向 | 方式 | 頻率 | 已確認 |
|---|---|---|---|---|
| 客戶 ERP（鼎新）| 雙向 | 鼎新 API | 進料 / 出貨即時、訂單 1hr | ✓ 規格確認 |
| LINE Notify | 出 | OAuth | 即時 | ✓ |
| 全家 / 超商客戶端 | 出 | 唯讀 API + QR | 隨需 | △ 等客戶確認規格 |
| 倉內 PLC（壓縮機）| 入 | Modbus TCP | 30s | ⚠ 待現勘確認型號 |

## 災難復原（DR）策略

| 等級 | 內容 | 我們採用 |
|---|---|---|
| Pilot Light | 跨 region 備援、災時擴展 | ✓ |
| Warm Standby | 縮小版備援 | — |
| Active-Active | 全規模並行 | — |

**理由**：食品冷鏈不是金融，分鐘級回應可接受，4hr RTO 用 Pilot Light 即可。
ap-northeast-1 主、ap-southeast-1 備（東京 → 新加坡）。

## 不可回滾情境

- 已對外發出的告警（LINE / Email）— 一旦客戶收到不能撤回
- 客戶已查詢過的履歷批號 — 對應的資料動到要走 CR + 客戶確認

## 系統範圍邊界

### 包含
- 冷凍倉 3 個 + 配送車 12 台 IoT 設備
- 中央雲端平台
- 員工後台、客戶查詢頁、司機 App
- 鼎新 ERP 整合
- 1 年運維支援

### 不包含
- 既有 ERP 改動（鼎新原廠負責）
- 廠區網路改善（客戶端網路商）
- 司機個人手機（用司機現有）
- 第二年起運維支援（另約）

## 反方審查預演

| 問題 | 我們的答案 |
|---|---|
| 為什麼用 AWS 不用地端？ | 6 個月上線、12 台車要 4G 跨地、地端建置 + 維運成本超過 6 個月雲費 |
| 凌晨網路斷怎麼告警？ | 邊緣告警（本地 LED + 蜂鳴）+ 4G 雙網路 |
| 食安法要 7 年我們存得起嗎？ | S3 Glacier、年成本 < 1 萬 |
| 上市櫃稽核時你能配合嗎？ | 合約寫含稽核配合 N 小時 |
| 李經理運維負擔？ | 第一年我方接手、第二年知識轉移、給 SOP |
