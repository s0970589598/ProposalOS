# AI 輔助提案流程

ProposalOS 把 AI 定位成**提案顧問**，不只是被提案物。

## AI 的雙重角色（很重要）

| 角色 | 對象 | 範例 |
|---|---|---|
| 被提案物 | 客戶 | 客戶系統裡的 AI 功能（如智慧告警） |
| 提案工具 | 我們 | Claude 幫我們寫提案、找漏洞、生內容 |

寫提案時要分清楚，**不要混在一起寫**。

## 提案各階段的 AI 用法

### 階段 1：產業研究

| AI 用法 | Prompt 重點 |
|---|---|
| 行業趨勢摘要 | 給 AI 行業名稱 + 3-5 個關鍵字，請他列趨勢 + 引用源 |
| 競品功能對比 | 給 AI 競品名單 + 對比維度，請他做表 |
| 痛點清單發想 | 給 AI 角色（如「製造業廠長」）+ 想知道的痛點維度 |

⚠️ AI 輸出的數字、廠商名稱、案例 **MUST 用 WebSearch / WebFetch 重驗**。不准直接寫進提案。

### 階段 3-4：訪談準備

| AI 用法 | Prompt 重點 |
|---|---|
| 生成訪談問題 | 給 AI 受訪者角色 + 想 cover 的痛點 |
| 訪談紀錄整理 | 把錄音逐字稿丟給 AI，請他整理成痛點清單 |
| 翻譯成 RTM | 把訪談重點丟給 AI，請他翻成需求 ID + 描述 |

### 階段 6：合規

| AI 用法 | Prompt 重點 |
|---|---|
| 法規影響分析 | 給 AI 產業 + 系統範圍，請他列適用法規 |
| ISO / NIST 對照 | 給 AI 我方架構，請他對照 NIST CSF 6 功能 |

⚠️ 法規引用要附法條編號，**MUST 重驗**。

### 階段 7：架構與方案

| AI 用法 | Prompt 重點 |
|---|---|
| 架構選項生成 | 給需求 + 限制，AI 列 2-3 種架構選項與優缺 |
| NFR 清單 | 給系統類型，AI 列 NFR 候選 |
| 風險清單 | 給架構，AI 列威脅模型 STRIDE |

### 階段 8：提案書產出

| AI 用法 | Prompt 重點 |
|---|---|
| 章節初稿 | 給 RTM + 架構，AI 寫章節初稿 |
| 改寫成客戶易懂 | 技術描述 → 業務語言 |
| 一致性檢查 | 找出前後矛盾、命名不一致 |

### 階段 9：反方審查

| AI 用法 | Prompt 重點 |
|---|---|
| 模擬法務質疑 | 給合約條款，AI 扮演客戶法務挑刺 |
| 模擬資安質疑 | 給資安章節，AI 扮演 CISO 提問 |
| 模擬老闆質疑 | 給效益章節，AI 扮演老闆 ROI 算盤 |
| 找漏洞 | 給整份提案，AI 對照 13 模組找缺漏 |

→ 詳見 [red-team-questions.md](red-team-questions.md)

## Prompt 模板

### 訪談紀錄 → 痛點清單

```
你是顧問。下面是客戶訪談逐字稿。請：

1. 找出 5-10 個明確痛點
2. 每個痛點用以下格式：
   - 痛點 ID（P-NN）
   - 角色（誰遇到）
   - 頻率
   - 影響（量化）
   - 客戶嘴上講的解法
   - 你判斷的真實問題（如果不同）

3. 不要編造數字。如果客戶沒講量化，標「未量化」。
4. 不要把客戶嘴上的解法當需求 — 區分「解法」與「問題」。

逐字稿：
[貼逐字稿]
```

### 反方審查

```
你是 [客戶法務 / CISO / 老闆]。
你的目標是把這份提案打回來。
請列出你會問的 10 個刁難問題，並指出提案中讓你不放心的地方。

不要客氣。不要鼓勵。盡量找漏洞。

提案：
[貼提案章節]
```

### 競品比較

```
給你 3 家競品：[A, B, C]，跟我們的方案。
請做一張對照表，維度包含：
- 功能完整度
- 產業契合度
- 價格區間（公開資訊或網路搜尋）
- License / 部署模式
- 客戶 case
- 認證

每個 cell 後面標 [verified / unverified / inferred]。
如果是 inferred，請另外列出來讓我手動驗。
```

## AI 用法的紅線

### ❌ 不能做

- 編造案例、客戶名單、引用源
- 編造法條編號、認證編號
- 編造數字（市場規模、客戶實績）
- 直接複製 AI 輸出進客戶提案
- 用 AI 取代訪談（AI 沒見過客戶）
- 客戶機敏資料丟到無 DPA 的 AI 服務

### ✅ 可以做

- 用 AI 起草、自己改寫
- 用 AI 找漏洞、自己判斷
- 用 AI 對照框架、自己決定取捨
- 用 AI 整理紀錄、雙方確認

## AI 輸出驗證流程

每次 AI 給內容，過三道關：

```
1. 事實核驗：數字、名稱、引用 → WebSearch / 既有 source 比對
   ↓
2. 邏輯檢查：論點是否成立、前後是否矛盾
   ↓
3. 語氣調整：客戶看了會怎麼想？太自信？太保守？
   ↓
寫進提案
```

## 工具選擇

| 工具 | 用途 |
|---|---|
| Claude Code | 主要提案工具、長文件操作 |
| Claude（網頁）| 短任務、討論 |
| 自家 RAG | 對接公司案例庫、SBOM、認證 |
| Web 搜尋 | 事實驗證 |

## AI 使用紀錄

每份提案要記錄：

- 哪些章節 AI 起草
- 哪些 AI 輸出有驗證
- 哪些是純人工
- AI 模型版本

未來客戶問「你是怎麼寫的」答得出來。
