# 客戶訪談腳本

從接觸到痛點清單的訪談流程。

## 訪談前準備

- [ ] 讀過客戶官網、年報、新聞（30 分鐘）
- [ ] 完成產業簡報（[modules/01](../framework/modules/01-industry-market.md)）
- [ ] 完成競品功課（[modules/02](../framework/modules/02-competitors-alternatives.md)）

- [ ] 準備錄音 / 記錄工具，並徵詢同意
- [ ] 準備 1 頁公司介紹（只在被問才拿出來）

## 產銷人發財 — 訪談覆蓋自我檢查

🔴 訪談**最容易踩的雷**：只挖到受訪者熟悉的 function，漏掉其他 function 的痛。
跟廠長談容易談「產」，但客戶真正缺的可能是「銷 / 財」的數據可見度。

每場訪談結束**自我檢查**：

| 字 | 業務功能 | 我問過嗎？ | 痛點筆記 |
|---|---|---|---|
| 產 | 生產 / 製造 / 交付 | [ ] | |
| 銷 | 銷售 / 行銷 / 通路 / 客服 | [ ] | |
| 人 | 人力 / 組織 / 訓練 | [ ] | |
| 發 | 研發 / 創新 / 產品開發 | [ ] | |
| 財 | 財務 / 會計 / 成本 / 資金 | [ ] | |

5 個都沒問到 → 安排第二輪訪談 / 換受訪者補。

→ 詳見 [模組 03 業務功能盤點](../framework/modules/03-problem-business-goals.md)

## 訪談類型

| 類型 | 對象 | 時長 | 目的 |
|---|---|---|---|
| 探勘訪談 | 提案發起人 / 老闆 | 60-90 min | 痛點、目標、決策 |
| 流程訪談 | 部門主管 | 60 min | As-Is 流程 |
| 第一線訪談 | 實際使用者 | 30-45 min | 真實痛點、抗拒源 |
| 技術訪談 | IT、資安 | 60-90 min | 整合、合規、技術限制 |
| 採購訪談 | 採購、財務 | 30 min | 採購流程、預算 |

## 五次「為什麼」結構

把客戶嘴上的解法問成真問題。

```
客戶：「我們要一個 AI 系統。」
你：為什麼想做 AI？
客戶：「我們報表很多。」
你：哪些報表？多久做一次？
客戶：「月報表，每月底要 3 天。」
你：3 天的時間花在哪？
客戶：「整理 Excel、跑 SQL、做圖、檢查。」
你：如果這 3 天變 1 天，對你最大的差別是什麼？
客戶：「我可以提早給老闆，他可以更早做月度決策。」

→ 真問題：月報產出時間 3 天，導致決策延遲
→ 解法不一定是 AI，可能是 Dashboard 自動化
```

## 探勘訪談腳本

### 開場（5 min）

```
- 自我介紹（30 秒）
- 訪談目的：「今天想了解貴公司目前狀況、想解的問題、評估過的選項」
- 錄音 / 記錄徵詢：「我會記錄重點，方便事後回顧。可以嗎？」
- 訪談紀錄會發給你確認：「我會在 24 小時內把紀錄發給你」
- 預期時長：「大約 60-90 分鐘」
- 你的問題優先：「中間有什麼想問都可以打斷」
```

### 背景了解（15 min）

```
1. 公司主要業務、規模？
2. 你的角色與職責？
3. 在這個職位幾年了？
4. 過去半年最大的挑戰是什麼？
5. 公司今年最重要的目標是什麼？
```

### 痛點探勘（30 min）

```
6. 為什麼想做這個系統？
7. 不做會怎樣？
8. 做了之後最希望改變什麼？

針對每個被提到的痛點，問：
- 誰遇到這個問題？（角色）
- 多久遇到一次？（頻率）
- 一次多痛？（影響量化）
- 現在怎麼處理？（變通方案）
- 試過什麼解決方案？結果？
- 為什麼還沒解？
```

### 已評估選項（15 min）

```
9. 之前評估過哪些供應商 / 工具 / 方案？
10. 為什麼沒選他們？
11. 自己做的選項有考慮嗎？多少預算 / 人力？
12. 維持現狀的選項呢？三年後會怎樣？
```

### 決策鏈（10 min）

```
13. 這個案子最後誰簽？
14. 還有誰會被諮詢？
15. 你預期的決策時程？
16. 過去類似案件的決策流程多久？
```

### 預算與時程（10 min）

```
17. 預算大概在什麼範圍？（給範圍讓對方挑）
18. 預算來源？（年度 / 專案 / 老闆）
19. 上線時程？有沒有 deadline？
20. 一定要在某個時間前上線嗎？為什麼？
```

### 收尾（5 min）

```
21. 還有什麼我們應該知道的？
22. 接下來會見哪些人？方便引介嗎？
23. 你希望我們下一步做什麼？
```

## 流程訪談腳本

針對 As-Is 流程：

```
1. 一個 [案件 / 訂單 / 案子] 從進來到結束，要經過哪些步驟？
2. 每個步驟誰負責？花多少時間？
3. 哪些步驟最常出錯？怎麼補救？
4. 哪些步驟靠某個特定的人？他請假怎麼辦？
5. 例外流程（客訴 / 退貨 / 緊急）怎麼跑？
6. 你最希望這個流程裡哪個環節變不一樣？
```

## 第一線訪談腳本

第一線最重要 — 他們是真正用系統的人。

```
1. 你每天的工作流程是什麼？
2. 你用哪些系統 / 工具？
3. 哪個工具讓你最痛苦？為什麼？
4. 如果可以許願改一件事，會是什麼？
5. 過去公司導入過系統嗎？感覺怎樣？
6. 對新系統最大的擔心是什麼？
```

## 技術訪談腳本

```
1. 既有架構 / 系統清單
2. 哪些有 API？哪些沒有？
3. 資料量級、頻率
4. 過去 12 個月的事故與處理
5. 內部資安政策（拿副本）
6. 雲端 vs 地端政策
7. 是否要 DPA / BAA
8. 認證要求（ISO、SOC 2、HIPAA）
9. 部署 window（什麼時候可以動）
10. 我們最該注意什麼？
```

## 訪談禁忌

- ❌ 開場直接推產品
- ❌ 急著回答客戶問題（先反問）
- ❌ 假設你聽到的解法等於需求
- ❌ 客戶講話打斷他
- ❌ 沒做紀錄
- ❌ 不確認紀錄就走
- ❌ 不問預算（怕問了顯得功利）
- ❌ 第一次訪談就報價

## 訪談後立刻做

- [ ] 整理紀錄、24 小時內發給受訪者確認
- [ ] 更新 RTM 痛點欄
- [ ] 更新角色 / 決策鏈
- [ ] 列下一步動作
- [ ] 內部 debrief（如有同事一起）
