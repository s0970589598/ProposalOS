# 產業加掛｜政府 / 教育 / 公共服務

中央 / 地方政府、學校、研究機構、公營事業。

## 對應 12 模組的擴充

### 模組 01 產業

- 政府數位轉型政策（國發會、數位部、地方）
- 智慧城市、SmartHigher Education
- 公共服務數位化（My Data、健保、稅）
- 多元族群（高齡、無障礙）

### 模組 02 採購

政府採購法**完全不同邏輯**：

| 議題 | 內容 |
|---|---|
| 採購法 | 政府採購法（公開招標、限制性招標、選擇性招標） |
| 採購門檻 | 公告金額、查核金額、巨額採購 |
| 議價空間 | 受限，要走法定程序 |
| 押標金 / 履約保證 | 必要、金額固定 |
| 評選方式 | 最有利標 / 最低標 |
| 反賄賂 | 嚴格、迴避利益衝突 |
| 政府電子採購網 | GPN 註冊、ePlatform |

### 模組 05 角色 — 政府特有

- 機關承辦人
- 採購委員會
- 評選委員（外部專家）
- 主計、政風
- 上級機關（部、署、教育部）
- 議會（地方層級）
- 監察、審計

### 模組 07 合規 — 政府必查

| 項目 | 內容 |
|---|---|
| 政府機關資通安全責任等級 | A 級 / B 級 / C 級 / D 級 |
| 資通安全管理法 | 各等級不同要求 |
| TWGCB（政府組態基準）| 系統設定基準 |
| 個資法 + 公務機關專章 | 較嚴 |
| 公文格式 / 簽核 | 公文系統整合 |
| 國產化要求 | 部分案件要求 |
| ESG / SDGs | 國際標案 |

### 模組 06 資料

政府資料特性：

| 項目 | 注意 |
|---|---|
| 國民身分證字號 | 嚴格保護、用途有限 |
| My Data 平台 | 跨機關介接的官方管道 |
| Open Data | 部分資料要開放 |
| 國家機密 | 機敏 / 密 / 機密 / 極機密 |

### 模組 08 架構

- 政府雲（GCB、TWGCB 環境）
- 信任應用清單
- 國產 vs 外商選擇
- 跨機關介接走 GSN

### 模組 11 驗收

- 嚴格按 SOW 驗收
- 延期 = 違約金（每日 0.1% 起跳）
- 履約保證金
- 文件交付（操作手冊、訓練、原始碼）

### 模組 12 智財

- **原始碼歸屬**：常被要求歸機關
- **GPL / 開源處理**：政府開源政策
- 委辦案 vs 採購案差異

## 政府特有議題

### 無障礙

- WCAG 2.1 AA / AAA
- 各機關網站無障礙標章
- 螢幕閱讀器、鍵盤操作

### 多語

- 中（繁）/ 英 必有
- 視機關加日 / 越 / 泰 / 客家

### 在地化

- 民國 / 西元曆
- 注音、台羅、客語
- 政府專屬簽章（自然人憑證、工商憑證）

### 公開透明

- 預算、決算、會議紀錄
- 個資去識別化後可公開
- 公文留檔 N 年

## 教育特有

- 校務系統（學籍、選課、成績）
- LMS（教學平台）
- 圖書館（OPAC、館際合作）
- 學術倫理（不可竄改、查重）
- 教育部統計回報
- 學生家長分權

## 反方審查問題

- 你符合政府機關資安責任等級 X 嗎？
- 採購法熟悉嗎？知道押標金多少嗎？
- 原始碼歸不歸機關？
- 無障礙符合 WCAG 哪一級？驗證過嗎？
- 政府雲還是私有雲？國產化要求？
- 延期罰金怎麼算？你扛得住嗎？
