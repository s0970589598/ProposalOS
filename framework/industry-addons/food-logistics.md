# 產業加掛｜食品 / 冷鏈 / 物流

食品製造、食品零售、餐飲、冷鏈、倉儲、物流。

## 對應 13 模組的擴充

### 模組 01 產業

- 食安風暴記憶猶新
- 冷鏈電商崛起
- 即時配送（30 分鐘到）
- 跨境食品電商
- 食品履歷、產銷履歷

### 模組 06 資料

| 資料 | 注意 |

### 模組 08 架構

冷鏈系統架構：

```
冷鏈現場（倉、車、櫃）
   ↓
溫度感測器 / GPS（1-15 min）
   ↓
Gateway / 4G 上傳
   ↓
雲端平台 + DB
   ↓
監控 / 告警 / 報表
   ↓
ERP / WMS / TMS 整合
```

### 模組 11 驗收

- 溫度紀錄完整率 100%
- 告警送達 SLA
- 追溯查詢時間（從出問題到鎖定批次幾分鐘）
- 客訴處理 SLA

## 食品 / 冷鏈特有議題

### 溫度監控不能斷

- 採樣間隔短（5-15 min）
- 上傳失敗要本地暫存
- 多備援（4G + WiFi + 衛星）
- 邊緣告警（網路斷時本地仍能告警）

### 食品召回

- 從出問題的消費者 → 反查批次 → 反查原料 → 鎖定影響範圍
- 召回時程要短（24-48h）
- 召回紀錄、政府通報

### 多溫層

- 常溫、冷藏（0-7°C）、冷凍（-18°C 以下）、深冷（-60°C）
- 不同溫層不同設備、不同告警閾值

### 物流即時性

- 即時 GPS、即時溫度
- 客戶查詢「我的訂單在哪」
- 司機 App、配送證明（POD）

### 跨境食品

- 進口報關
- 各國食品安全法規
- 標示翻譯
- 海關抽驗

## 強制檢查項加強

### A RTM

要把「批次追溯」當核心需求拉出來

### F RTO / RPO

- 溫度資料 RPO ≈ 0（不能遺失）
- 系統停機影響食安要寫罰金 / 賠償

### G 退出機制

- 食品履歷資料屬法定保存
- 留存期 N 年（依產品 5-10 年）

## 反方審查問題

- 溫度斷鏈 30 分鐘，你怎麼告警 / 怎麼處理？
- 一批產品出問題，你多久能找出影響範圍？
- 我們有 3 個冷鏈溫層，告警邏輯能客製嗎？
- 食品履歷上政府平台你做過嗎？
- 召回演練做過嗎？
- 配送 GPS 跟溫度怎麼整合到一張圖？
