# 產業加掛｜金融

銀行、保險、證券、信用卡、FinTech、Crypto。

## 對應 12 模組的擴充

### 模組 01 產業

- Open Banking / 純網銀 / 純網保
- 監理沙盒、TWQR
- 反洗錢、反恐怖融資（AML / CFT）
- 區塊鏈、CBDC、虛擬資產

### 模組 02 採購

金融業採購規範嚴：

- 委外作業辦法（金管會）
- 重要委外要事前申報
- 第三方供應商風險評估
- 母公司核准（跨國集團）

### 模組 05 角色 — 金融特有

- 法令遵循（Compliance）
- 風險管理（Risk Management）
- 內稽內控
- 資安長（CISO）
- 法務
- 金融監理單位（金管會、央行）

### 模組 07 合規 — 金融必查

| 項目 | 內容 |
|---|---|
| 金管會委外作業辦法 | 委外條件、雲端委外 |
| 銀行業電腦稽核制度 | 系統稽核 |
| 個資法 + 金融機構辦法 | 較嚴 |
| PCI-DSS | 信用卡資料 |
| AML / CFT | 反洗錢 |
| SWIFT CSP | 國際匯款 |
| ISO 27001 / SOC 2 | 必有 |
| 公平交易 | 行銷、利率 |
| 證券業 / 保險業專屬法規 | 視業別 |

### 模組 06 資料

| 議題 | 內容 |
|---|---|
| 客戶資料 | 嚴格分級、查閱稽核 |
| 交易資料 | 完整性、不可篡改 |
| 跨境傳輸 | 受監理、要報准 |
| 雲端委外 | 雲端區域限制（金管會） |
| 資料留存 | 7-10 年 |

### 模組 08 架構

| 項目 | 內容 |
|---|---|
| 上下行隔離 | DMZ、防火牆分層 |
| 高可用性 | 99.99%+、Active-Active |
| 災難復原 | 異地、定期演練 |
| 加密 | At-rest、In-transit、HSM |
| 金鑰管理 | HSM、KMS |
| 稽核日誌 | 不可篡改、長期保存 |

### 模組 11 驗收

- 內稽 / 內控 / 外稽要過
- UAT 加入合規測試
- Pen test 第三方執行
- 上線需主管機關核准（部分業務）

### 強制檢查項 D Secure SDLC

金融加強要求：

- Code Review 雙人覆核
- 生產環境變更必經 CAB
- 機敏資料存取 4-eye
- 客戶資料禁止下載到開發環境

### 強制檢查項 E SBOM

金融嚴格管控：

- 所有第三方元件報准
- 重要元件 escrow
- 漏洞修補 SLA 嚴格（Critical 24h）

## 金融特有議題

### 雲端委外

- 金管會「金融業辦理電子作業委外處理作業辦法」
- 重要委外事前核准
- 雲端區域限制（資料是否可離境）
- 雲端供應商盡職調查

### KYC / AML

- 開戶身份驗證
- 持續監控
- 可疑交易通報（STR）
- 制裁名單檢查（OFAC、台灣外交部）

### 高可用 / 高效能

- 99.99%+ 可用率
- 交易延遲毫秒級
- 災難復原 RTO < 4h、RPO < 15min（核心）

### 稽核軌跡

- WORM 儲存
- 集中日誌、無法本機刪除
- 異常告警直送 SOC

- 災難演練多久一次？上次什麼時候？
