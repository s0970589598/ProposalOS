# 產業加掛｜零售 / 電商 / 餐飲 / 連鎖

實體零售、電商、O2O、餐飲、連鎖店、批發。

## 對應 13 模組的擴充

### 模組 01 產業

- 全通路（OMO / Omnichannel）
- 會員經濟、訂閱制
- 直播電商、社群電商
- 跨境電商、虛實整合
- 物流最後一哩

### 模組 03 問題

| 場景 | 痛點 |

| 庫存 | 多通路 / 多倉、即時性差 |

| 金流 | 多支付、對帳、退款 |

## 零售特有議題

### 高峰應對

- 雙 11、黑五、母親節等
- 流量可能 10-100 倍正常值
- 容量規劃、限流、降級

### 多通路訂單

- 同一會員在不同平台
- 訂單統一視圖
- 對帳、退款、會員權益

### 行銷自動化

- 觸發式 EDM / SMS / LINE
- 棄購挽回
- 個人化推薦

### 物流

- 多家物流 API（黑貓、新竹、宅配通、超商）
- 國際物流（DHL、FedEx）
- 最後一哩、配送追蹤

### 反詐欺

- 信用卡盜刷
- 黃牛、刷單
- 退款濫用

## 反方審查問題

- 雙 11 流量 50 倍，你的系統撐得住嗎？實測過？
- 多通路會員拼接準確率多少？
- 庫存超賣怎麼防？
- 退款流程怎麼自動化？
- 第三方支付串接幾家？failover 怎麼處理？
- PCI-DSS 等級多少？範圍包含什麼？
