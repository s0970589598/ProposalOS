# Business Model Canvas + Value Proposition Canvas

Alexander Osterwalder 提出（2010 / 2014）。
**訪談客戶生意的高端工具**。讓客戶**自己畫他的 BMC**、等於免費顧問。

## Business Model Canvas（BMC）— 9 格

```
┌─────────────┬─────────────┬─────────────┐
│ Key Partners│ Key         │ Value       │
│             │ Activities  │ Propositions│
│             ├─────────────┤             │
│             │ Key         │             │
│             │ Resources   │             │
├─────────────┴─────────────┼─────────────┤
│ Channels                  │ Customer    │
│                           │ Relationships│
│                           ├─────────────┤
│                           │ Customer    │
│                           │ Segments    │
├─────────────────────────────────────────┤
│ Cost Structure   │ Revenue Streams      │
└─────────────────────────────────────────┘
```

9 格：

| 格 | 問題 |
|---|---|
| Customer Segments | 我服務誰？ |
| Value Propositions | 我給他什麼價值？ |
| Channels | 我怎麼接觸他們？ |
| Customer Relationships | 怎麼建關係 / 留客 |
| Revenue Streams | 我怎麼賺錢？ |
| Key Resources | 我需要什麼資源 |
| Key Activities | 我做什麼活動 |
| Key Partners | 誰是我的合作夥伴 |
| Cost Structure | 我花什麼錢 |

## 用 BMC 訪談客戶

訪談時帶白板、跟客戶**一起畫他公司的 BMC**：

```
明陽食品 BMC（示意）：

Customer Segments：B2B 餐飲通路（全家、便利商店、超市）
Value Propositions：穩定品質、即時配送、客製規格
Channels：直接業務、ERP 串接
Revenue Streams：產品銷售（800 萬/月）
Cost Structure：原料 60%、人力 20%、物流 10%
Key Resources：3 個冷凍倉、12 配送車、品管團隊
Key Activities：採購、加工、配送、品管
Key Partners：原料供應商、冷凍車保養廠、ERP 廠商
```

→ 客戶**看到自己生意全貌**、會自己發現痛點：
- 「我們 Key Activities 沒寫『稽核』，但其實花很多時間」
- 「Value Proposition 包含『穩定品質』，但我們其實常斷鏈影響穩定」

→ 順便讓客戶看到**我們的提案會影響哪幾格**：

```
明陽 BMC + GoodLinker 影響：

Value Propositions：穩定品質 ← GoodLinker 強化
Key Activities：品管、稽核 ← GoodLinker 自動化
Key Resources：冷鏈系統 ← GoodLinker 新增
Cost Structure：客訴賠款 ← GoodLinker 降低
```

→ **比直接報價更有說服力**（客戶看到我們是「整體生意的一塊」、不只是供應商）。

## Value Proposition Canvas（VPC）— BMC 的放大版

聚焦其中兩格：Customer Segments + Value Propositions。

```
       Customer Profile（客戶輪廓）        Value Map（價值地圖）

       ┌─────────────────────────┐         ┌─────────────────────────┐
       │   Gains（益處）          │         │   Gain Creators          │
       │                          │         │                          │
       │      ┌──────┐            │   →←    │            ┌──────┐     │
       │      │      │            │         │            │      │     │
       │      │ Jobs │            │         │  Products  │      │     │
       │      │      │            │         │  Services  │      │     │
       │      └──────┘            │         │            └──────┘     │
       │                          │         │                          │
       │   Pains（痛點）          │         │   Pain Relievers         │
       └─────────────────────────┘         └─────────────────────────┘
```

### Customer Profile 三角

| 元素 | 內容 |
|---|---|
| Jobs | 客戶想完成的工作（JTBD）|
| Gains | 想要的好處 |
| Pains | 想避免的壞處 |

### Value Map 三角

| 元素 | 對應客戶端 |
|---|---|
| Products & Services | 我提供什麼 |
| Gain Creators | 我怎麼創造客戶的 Gains |
| Pain Relievers | 我怎麼解決客戶的 Pains |

### Fit（契合）

兩邊三角對齊 = **Product-Market Fit**。
提案要顯示**我們的 Value Map 對應客戶 Profile**。

## 跟 ProposalOS 整合

### 訪談階段（階段 4）加 BMC 工作坊

| 時間 | 活動 |
|---|---|
| 30 min | 帶客戶填 BMC 9 格 |
| 20 min | 找出哪幾格是我們會影響到的 |
| 30 min | 用 VPC 對齊「我們價值 vs 客戶痛點 / 益處」 |

### 模組 03 問題定義 加 VPC

把痛點映到 Customer Profile 的 Pains、把方案映到 Value Map 的 Pain Relievers。

### 模組 09 方案 加 Gain Creators 視角

不只解 pain（必備）、也要顯示我們有 Gain Creators（驚喜，對應 Kano Delight）。

## 適用情境

| 情境 | BMC / VPC 適用 |
|---|---|
| 客戶老闆親自參與 | ✅ 高（高層喜歡這種戰略視角）|
| 部分功能型 / 工具型提案 | △ 部分（不必整套畫）|
| 純技術更新案件 | ❌ 過度設計 |
| 跨產業多功能整合提案 | ✅ 必用（顯示我們懂整體）|

## 常見錯誤

- ❌ 拿 BMC 模板自己填、沒帶客戶一起 → 失去訪談價值
- ❌ 填完不串到提案 → 變展示用
- ❌ 用 VPC 寫得很美、但 fit 不真實 → 客戶看穿
