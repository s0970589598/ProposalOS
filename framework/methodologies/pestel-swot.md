# PESTEL + SWOT

兩個經典戰略分析工具。
**PESTEL** 看外部環境、**SWOT** 看內部+外部交叉。

## PESTEL — 外部環境分析

```
P  Political      政治：政府、政策、法規方向
E  Economic       經濟：景氣、利率、匯率、消費力
S  Social         社會：人口、文化、價值觀、生活方式
T  Technological  科技：新技術、研發、自動化、AI
E  Environmental  環境：氣候、永續、ESG、地震洪水
L  Legal          法律：法規變動、合規、稅務
```

## PESTEL 在提案的應用（模組 01 產業）

每個案件對應 6 字、找出趨勢：

### 明陽食品 PESTEL 範例

| 字 | 範例 |
|---|---|
| **P** 政治 | 食安法 2024 修正、政府推食品追溯、台美貿易 |
| **E** 經濟 | 通膨、能源價漲、餐飲業復甦 |
| **S** 社會 | 食安意識提升、消費者要求透明 |
| **T** 科技 | IoT 成本降、AI 預測成熟、5G 普及 |
| **E** 環境 | ESG 報告必交、淨零、CBAM |
| **L** 法律 | 食安法、個資法、上市櫃永續準則 |

→ 每個案件**至少寫 3-5 個外部驅動因子**、強化「為什麼是現在做」。

## SWOT — 內部 + 外部交叉

```
            內部                外部
       ┌────────────┬────────────┐
   正面 │ Strengths  │ Opportunities│
       │   優勢      │    機會      │
       ├────────────┼────────────┤
   負面 │ Weaknesses │ Threats     │
       │   劣勢      │    威脅      │
       └────────────┴────────────┘
```

| 維度 | 內容 |
|---|---|
| Strengths（優勢）| 我們有什麼別人沒有 |
| Weaknesses（劣勢）| 我們缺什麼 |
| Opportunities（機會）| 外部有什麼可抓 |
| Threats（威脅）| 外部有什麼威脅 |

## SWOT 提案中三種用法

### 1. 對「我方公司」做 SWOT（內部用、別給客戶看）

| | 內部 |
|---|---|
| S | 同產業 8 個 case、團隊穩定、ISO 27001 |
| W | 公司規模小、外商認證少、IT 人才稀 |

| | 外部 |
|---|---|
| O | 食安法強化、客戶 ESG 壓力、上市櫃潮 |
| T | 國際大廠下沉、客戶 IT 自建潮、AI 取代部分服務 |

→ 我方 SWOT 用來內部判斷 Bid/No-Bid、決定 Win Themes。

### 2. 對「客戶」做 SWOT（訪談用、跟客戶一起做）

跟客戶一起做客戶自己公司的 SWOT、等於免費顧問。
順便發現「我們能幫上的點」。

### 3. 對「競品」做 SWOT（模組 02 競品分析）

| 競品 | S | W | O | T |
|---|---|---|---|---|
| 國際大廠 B | 品牌、認證 | 太貴、客製差 | 全球 IoT 趨勢 | 在地化挑戰 |
| 純 IoT 商 A | 便宜、快 | 沒後台、無服務 | 中小企業預算緊 | 整合複雜需求 |
| 我方 | 中型客戶經驗、貼台灣 | 規模小 | 食安法 | 大廠下沉 |

→ 從 SWOT 找出我方**Discriminator**（Win Themes）：「中型客戶經驗 + 貼台灣食安法 + 完整後台 + 合理價格」。

## SWOT 進階：TOWS 矩陣

用 SWOT 結果產生戰略：

| | Strengths | Weaknesses |
|---|---|---|
| **Opportunities** | SO 戰略：用優勢抓機會 | WO 戰略：克服劣勢抓機會 |
| **Threats** | ST 戰略：用優勢避威脅 | WT 戰略：減少劣勢避威脅 |

範例：

| 象限 | 戰略 |
|---|---|
| SO | 用我方食品經驗 + 食安法強化趨勢 → 主打中型食品客戶 |
| WO | 缺外商認證 + 客戶 ESG 壓力 → 補強 ESG 報告功能 |
| ST | 用同產業 case + 大廠下沉 → 強調「在地深度 > 全球廣度」 |
| WT | 規模小 + 客戶自建潮 → 強調我方「服務 + 顧問」非純產品 |

## 跟 ProposalOS 整合

### 模組 01 產業市場 加 PESTEL

讓「產業趨勢」段落更結構化、不會只寫一句「智慧製造起來」。

### 模組 02 競品 加 SWOT（每個競品）

讓競品分析更立體、不只是功能對照。

### 模組 12 風險 加 ST / WT 戰略

從威脅 → 對應風險登記、有對策。

### Capture Plan 必有（Shipley 方法）

[Shipley / APMP](shipley-apmp.md) 的 Capture Plan 內含 SWOT 對自己 + 對手 + 客戶。

## 適用情境

PESTEL：
- 中型以上案件 → 必做
- 小型工具型案件 → 可省

SWOT：
- 任何案件 → 內部 Bid/No-Bid 時做
- 提案文件中 → 可隱去自己的 W/T、只給客戶看 S/O

## 常見錯誤

- ❌ PESTEL 寫得空泛、每字一句廢話
- ❌ SWOT 只寫優勢、不誠實看劣勢
- ❌ 把 SWOT 當清單、沒做 TOWS 戰略推導
- ❌ 對客戶亮自己的 W / T（自爆）

## 推薦讀物

- **SWOT 起源**：Albert Humphrey 1960s 在 Stanford Research Institute (SRI) 開發、後續 Learned/Christensen/Andrews/Guth 在 Harvard 1965 形式化
- **PESTEL 演化**：原是 PEST、後續加上 Environmental 和 Legal 變 PESTEL（也作 STEEPLE / STEEPLED）
- Michael Porter《Competitive Strategy》（1980、Five Forces，跟 SWOT 是**不同框架**但常一起用）
- HBS Case Method 中 SWOT / PESTEL 應用
- 商業評論常見「五力分析 + PESTEL + SWOT」三角組合

## 跟其他方法的關係

| 方法 | 跟 PESTEL/SWOT |
|---|---|
| Porter's Five Forces | 看產業競爭、跟 PESTEL 互補 |
| Wardley Mapping | 看演化階段、SWOT 補時間維度 |
| BCG Matrix | 看產品組合 |
| Ansoff Matrix | 看成長戰略 |
| OKR | SWOT 找方向、OKR 落地 |
