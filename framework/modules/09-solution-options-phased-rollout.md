# 模組 09｜解決方案選項與分階段導入

🔴 必做

## 目的

不要只給一個方案。給 2-3 個方案讓客戶選，並把分階段路線講清楚。

## 為什麼要多方案

| 給一個方案 | 給 2-3 個方案 |
|---|---|
| 「要 / 不要」二選一 | 「選哪個」三選一 |
| 客戶可能不買 | 客戶會掉進選哪個 |
| 顯不出我們的專業 | 顯示我們懂取捨 |
| 無法談判 | 可拆組合、可加減 |

## 三方案模板

| 方案 | 定位 | 範圍 | 時程 | 價格區間 |
|---|---|---|---|---|
| **基礎版** | 解決核心痛點 | 模組 08 的 P0 | 短（2-3 個月） | 低 |
| **標準版** | P0 + 部分 P1 | 加報表、整合 | 中（3-6 個月） | 中 |
| **完整版** | P0 + P1 + 部分 P2 | 加 AI、進階分析、自動化 | 長（6-12 個月） | 高 |

每個方案要寫：

- 包含什麼功能（對應 FR-001 ～ FR-XXX）
- **不**包含什麼（明確排除）
- 預期效益（量化）
- 風險（每個方案的特有風險）
- 適合什麼樣的客戶 / 階段

## 分階段導入（Phased Rollout）

不論選哪個方案，都要分階段：

```
階段 1：基礎建設     │ 月 1-2 │ 環境、整合、最小可用
階段 2：核心功能上線 │ 月 3-4 │ P0 功能、Pilot 部門
階段 3：擴大採用     │ 月 5-6 │ 全公司、訓練、調整
階段 4：優化迭代     │ 月 7+  │ P1、KPI 追蹤、報告
```

### 每階段必寫

- **目標**：這階段做完要達到什麼狀態
- **可交付物**：具體可驗收的產出
- **里程碑**：客戶要做什麼決定 / 簽核
- **退出條件**：什麼狀況下這階段算完成
- **下一階段啟動條件**：什麼條件達成才進下一階段

## 常見地雷

- ❌ 三方案功能差不多，只是價格不同 → 客戶看不出差異
- ❌ 全部一次上線 → 風險爆炸
- ❌ 沒寫「不包含」 → 範圍蔓延
- ❌ 階段間沒退出條件 → 卡住沒人敢往前
- ❌ 全部用 Big Bang 上線 → 失敗一次全廠停工

## 分階段策略選擇

| 策略 | 適用 | 風險 |
|---|---|---|
| Pilot Department | 大型組織 | 部門差異大時擴散難 |
| Pilot Region | 多地點 | 跨地整合複雜 |
| Pilot Feature | 功能漸進 | 多次切換成本 |
| Parallel Run | 高風險 | 成本高、人員負擔 |
| Big Bang | 小型、簡單 | 失敗影響大 |

## 輸出物

1. **三方案對照表**：功能、時程、價格、效益、風險
2. **推薦方案說明**：為什麼推薦這個（不能不推）
3. **分階段路線圖**：每階段目標、產出、里程碑
4. **退出條件表**：每階段過關標準
5. **客戶選擇樹**：什麼情況選哪個

## 與其他模組的關係

- ← 模組 02：競品分析 → 我們方案的差異定位
- ← 模組 08：功能、NFR → 方案範圍切分
- → 模組 10：每方案對應一份報價
- → 模組 11：每階段對應一次驗收

## 反方審查問題

- 為什麼是這三個方案？不是五個？
- 你推薦哪個？為什麼？
- 客戶選便宜版，未來升級會卡嗎？
- 階段 2 失敗，可以退回階段 1 嗎？資料怎麼辦？
- 客戶現在的選擇，跟一年後再決定，差多少？
