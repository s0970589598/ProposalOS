# 模組 07｜ISO、法規、資安、個資與 AI 風險

🔴 必做（不同產業深淺不同）

## 目的

法規、資安、AI 是現在客戶法務 / IT / 採購最常擋的三個點。
不能等到簽約前才補。

## 四個面向

### 1. 法規與標準

| 類別 | 常見項目 | 觸發產業 |
|---|---|---|
| 通用資安 | ISO 27001、SOC 2、NIST CSF 2.0 | 大客戶、跨國、金融 |
| 個資保護 | 個資法、GDPR、CCPA | 所有 to-C、跨國 |
| 產業特定 | HIPAA（醫療）、PCI-DSS（支付）、TISAX（車用） | 對應產業 |
| 品質管理 | ISO 9001、IATF 16949 | 製造、車用 |
| 環境永續 | ISO 14001、ESG、碳盤查 | 上市櫃、出口、製造 |
| AI 治理 | NIST AI RMF、EU AI Act、ISO 42001 | 用 AI 的任何案件 |

### 2. 資安（用 NIST CSF 2.0 六大功能對齊）

| 功能 | 內容 | 提案要寫什麼 |
|---|---|---|
| Govern（治理） | 資安治理、政策、角色 | 誰是 owner、政策來源 |
| Identify（識別） | 資產、風險、威脅模型 | 資產清單、風險矩陣 |
| Protect（保護） | 存取控制、加密、訓練 | 權限模型、加密方式 |
| Detect（偵測） | 日誌、監控、異常告警 | 監控覆蓋率、告警規則 |
| Respond（回應） | 事件分級、通報、止血 | 事故處理流程、通報窗口 |
| Recover（復原） | 備份、還原、業務持續 | RTO / RPO、演練頻率 |

### 3. 個資與資料保護

| 項目 | 必寫內容 |
|---|---|
| 蒐集範圍 | 蒐集哪些個資、目的、法源 |
| 同意機制 | 怎麼取得同意、可否撤回 |
| 存取控制 | 誰能看、能改、能匯出明文 |
| 加密 | 靜態加密、傳輸加密、金鑰管理 |
| 脫敏 | 哪些場景用脫敏資料 |
| 保留期 | 多久刪除、怎麼刪 |
| 跨境傳輸 | 資料是否離境、合法基礎 |
| 個資事故 | 通報流程、72 小時通報（GDPR） |
| 當事人權利 | 查閱、更正、刪除、攜出 |

### 4. AI 風險（依 NIST AI RMF）

| 維度 | 內容 |
|---|---|
| Govern（治理） | AI 使用政策、責任歸屬 |
| Map（盤點） | 哪些功能用 AI、輸入輸出是什麼 |
| Measure（衡量） | 準確率、偏見、可解釋性 |
| Manage（管理） | 監控、回饋迴路、版本控管 |

AI 額外要寫：
- **模型版本管理**：用哪個模型、版本、何時更換
- **Prompt 管理**：Prompt 怎麼版控、誰能改
- **輸出評估**：怎麼判斷 AI 輸出對不對
- **人工覆核**：哪些場景必須人工覆核
- **錯誤責任**：AI 出錯誰負責、怎麼補救
- **資料使用**：客戶資料會不會被拿去訓練

## 常見地雷

- ❌ 「我們符合 ISO 27001」沒附證書 → 客戶法務會要
- ❌ 「資料加密」沒寫演算法 / 金鑰管理 → 等於沒寫
- ❌ 「AI 很準」沒寫評估方式 → 客戶會問怎麼驗
- ❌ 沒寫個資 sub-processor → GDPR 案件會卡
- ❌ 假設客戶沒有跨境 → 越南 / 馬來西亞分公司就有跨境

## 必問問題

- 客戶有沒有 ISO 27001 / SOC 2 要求？什麼時候要？
- 客戶的資料會不會跨境？哪些國家？
- 客戶內部資安 / 法務怎麼運作？要簽什麼文件（DPA、BAA）？
- 客戶對 AI 的態度？有沒有 AI 使用政策？
- 過去有沒有資安事故？學到什麼？

## 輸出物

1. **合規矩陣**：每條法規 / 標準 → 對應的提案章節
2. **資安架構說明**：用 NIST CSF 6 功能組織
3. **個資保護聲明**：蒐集、使用、保留、刪除、跨境
4. **AI 治理說明**（若有 AI）
5. **責任邊界**：我們負什麼、客戶負什麼

## 反方審查問題

- 客戶法務一問「DPA 在哪」你拿得出來嗎？
- 「資料加密」客戶問演算法、金鑰管理、輪替頻率你答得出來嗎？
- AI 出錯責任誰擔？合約有寫嗎？
- 你有沒有要求過客戶提供他們的資安政策？
- 跨境傳輸的合法基礎是什麼？
