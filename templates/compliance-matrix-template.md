## 必要性分級

| 字 | 意思 | 不符後果 |
|---|---|---|
| **強制 / 必須 / 應 / shall / must** | 一定要 | 直接 disqualify |
| **應該 / should** | 應該要 | 扣分 |
| **得 / 可以 / may** | 可以選做 | 不扣分 |

⚠️ **「應」在台灣政府採購法實務 = 「shall」、是強制義務、不是軟性建議**。
雖然中文「應」字面感覺較軟、但**法律上 = 必須**、漏 1 條 disqualify。

⚠️ 「應該」才是軟性 should。兩字差一個字、後果差很多。

⚠️ 有疑問先在投標前 Q&A 階段問清楚、**不要假設「應」是軟性**。

## 對應程度分級

| 級 | 意思 |
|---|---|
| **完全符合**（Fully Compliant）| 100% 達標、有證明 |
| **部分符合**（Partially Compliant）| 達到部分要求、附對策 |
| **不符合**（Non-Compliant）| 達不到、要在備註說明替代方案 |
| **超出要求**（Exceeds）| 高於要求、強調差異化 |

---

## 缺口處理策略

### 對「部分符合」項

1. 在備註欄詳細說明為什麼是部分
2. 提供替代方案 / 補強計畫
3. 不要試圖隱藏 → 評委發現會扣更多

### 對「不符合」項

決策：
- 完全做不到 → No-Bid
- 可以做但要時間 → 提案中標「將於 X 個月內補強」
- 規格不合理 → Q&A 階段提出
- 我方有更好方案 → 提案中說明差異

## 自我檢查清單

提案送出前：

- [ ] 每條招標規格都有對應行
- [ ] 沒有任何「強制 / 必須」標為「不符合」
- [ ] 每個「完全符合」都有證明文件
- [ ] 證明文件已附於提案附錄
- [ ] 「部分符合」都有對策說明
- [ ] 經內部法務複核

## 常見地雷

| 地雷 | 後果 |
|---|---|
| 漏 1 條強制條款 | Disqualify |
| 證明文件過期 | 視為不符 |
| 「部分符合」沒寫對策 | 評委質疑 |
| 「完全符合」沒附證明 | 被質疑造假 |
| 用「同等規格」沒解釋 | 不被接受 |
| 押標金面額錯 | 投標無效 |
