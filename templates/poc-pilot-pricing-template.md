# PoC / Pilot Pricing 範本

⚠️ **不是所有案件都該一次報「整套」**。
複雜 / 創新 / Cynefin Complex 情境 → 用 PoC / Pilot 階段定價。

對應 [Cynefin Framework](../framework/methodologies/cynefin.md)。

---

## 何時用 PoC / Pilot Pricing

| 情境 | PoC/Pilot 適用 |
|---|---|
| 客戶要做新領域（如新 AI 應用）| ✅ |
| 客戶內部對技術方向不確定 | ✅ |
| 預算不確定但想試 | ✅ |
| 規格已定、要全部一次做 | ❌（用正常 RFP / RFQ） |
| 客戶不想投入大資源評估 | ✅ 入門價 |
| 客戶已決定買、只比價 | ❌ |

---

## 三層階梯定價模型

```
PoC（驗證概念）
   ↓
Pilot（小規模試行）
   ↓
Production（正式上線）
   ↓
Expansion（擴大）
```

### PoC（Proof of Concept）

| 項目 | 內容 |
|---|---|
| 目的 | 驗證**技術可行性** |
| 期間 | 4-8 週 |
| 範圍 | 1 個核心功能、少量資料 |
| 結果 | 報告 + Demo，**不上 prod** |
| 預算 | NT$ 30-100 萬 |
| 成功標準 | 雙方事前同意 |

### Pilot（試行）

| 項目 | 內容 |
|---|---|
| 目的 | 驗證**業務價值** |
| 期間 | 2-3 個月 |
| 範圍 | 小規模真實使用（1 個倉 / 1 部門 / 1 個產品線）|
| 結果 | KPI 改善數據 |
| 預算 | NT$ 100-300 萬 |
| 成功標準 | 量化 KPI 達標 |

### Production（正式）

| 項目 | 內容 |
|---|---|
| 目的 | 全面上線 |
| 期間 | 3-12 個月 |
| 範圍 | 全公司 / 全廠 |
| 結果 | 完整系統 |
| 預算 | NT$ 300 萬+ |
| 成功標準 | 完整 SLA |

## 報價範本

### 情境：明陽食品 AI 配送預測（Cynefin Complex 案例）

```
階段 1：PoC（4-8 週）
├── 範圍：用過去 6 個月歷史資料、訓練配送預測模型
├── 不含：實際接入即時系統、上線
├── 預算：NT$ 80 萬
├── 付款：簽約 50% / 完成 50%
└── 成功標準：模型 MAE < 15%、客戶 IT 認可

階段 2：Pilot（3 個月）
├── 觸發：PoC 通過、雙方同意進入 Pilot
├── 範圍：對 3 條配送路線實際使用、人工平行驗證
├── 不含：全車隊
├── 預算：NT$ 200 萬
├── 付款：分 3 段
└── 成功標準：3 個月 KPI 改善 ≥ 10%

階段 3：Production（6 個月）
├── 觸發：Pilot 達標、董事會核可
├── 範圍：全 12 台車、完整整合
├── 預算：NT$ 600 萬
├── 付款：簽約 30% / 階段交付 40% / 驗收 30%
└── 成功標準：完整 SLA、12 個月 KPI 達標

階段 4：Expansion
├── 觸發：Production 上線 6 個月後評估
├── 可能擴展：第二廠、AI 改良、新功能
└── 預算：另議
```

---

## 階段間的決策點

每階段結束、要明確**繼續 / 暫停 / 終止**：

| 決策 | 觸發條件 | 動作 |
|---|---|---|
| 繼續 | 達標 | 簽下一階段合約 |
| 暫停 | 部分達標、需要調整 | 修改範圍重新議 |
| 終止 | 未達標、不可行 | 結算、雙方退出 |
| 跳過 | PoC 結果太好、直接 Pilot+Production | 重議合約 |

⚠️ 客戶常見質疑：「萬一 PoC 過了你獅子大開口怎麼辦？」
→ 解：**Production 上限價在 PoC 階段就鎖**、雙方寫進 MOU。

## PoC 合約條款範本

```
1. 範圍
   1.1 PoC 範圍如附件 A
   1.2 不包含：實際 prod 部署、即時系統整合、客戶端使用者訓練

2. 期間
   8 週，可雙方同意延長 1 次

3. 費用
   NT$ 80 萬，分 2 段付款（50% 簽約 / 50% 完成）

4. 成功標準
   4.1 模型在保留資料集（hold-out set）上 MAE < 15%
   4.2 雙方共同確認的 demo 場景全部通過
   4.3 PoC 結束報告經客戶 IT 主管簽核

5. PoC 結果歸屬
   5.1 客戶取得 PoC 報告、demo、訓練好的模型權重
   5.2 我方保留方法論、工具
   5.3 客戶資料保持客戶所有

6. 後續合作
   6.1 雙方互不承諾必須進入 Pilot / Production
   6.2 但雙方同意 Pilot 報價上限 NT$ 300 萬、Production 上限 NT$ 600 萬
   （上限數字以本範本前段工作範例為準、實際合約以雙方確認為主）

7. 智財
   7.1 PoC 過程產生的客製化程式碼歸客戶
   7.2 通用方法、工具、平台核心歸我方

8. 退出
   PoC 結束 30 天內、若雙方未進入下階段、所有客戶資料於 60 天內刪除並出具證明
```

## 適用客戶

| 客戶類型 | PoC/Pilot 適用度 |
|---|---|
| 創新 / R&D 部門案件 | ✅ 高 |
| AI / ML 案件 | ✅ 高 |
| 老闆「想試試看」 | ✅ |
| 客戶已確定要做整套 | ❌ 反而拖延 |
| 政府公開招標 | ❌ 通常規格已定 |

## 常見錯誤

- ❌ PoC 範圍太大、變成迷你 Production
- ❌ 沒明確成功標準、PoC 結束爭執
- ❌ Production 上限價沒鎖、客戶覺得被綁
- ❌ PoC 失敗就無下文、沒有「暫停」選項
- ❌ PoC 階段送 demo 給其他部門看（範圍蔓延）
