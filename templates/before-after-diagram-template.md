# 導入前後差異圖 範本

## 目的

讓老闆 / 主管 30 秒看懂：以前怎樣 → 以後怎樣 → 差多少。

## 三種表達方式

### 方式 1：左右並列圖（推薦給簡報）

```
┌──────────── 導入前 ────────────┐    ┌──────────── 導入後 ────────────┐
│                                 │    │                                 │
│  [As-Is 流程圖]                  │ →  │  [To-Be 流程圖]                  │
│                                 │    │                                 │
│  - 人工抄表                      │    │  - 自動採集                      │
│  - Excel 整理                    │    │  - 即時 dashboard                │
│  - 月報 3 天                     │    │  - 即時報表                      │
│  - 異常 3h 後知                  │    │  - 異常 5min 告警                │
└─────────────────────────────────┘    └─────────────────────────────────┘
```

### 方式 2：對照表

| 維度 | 導入前 | 導入後 | 改善 |
|---|---|---|---|
| 巡檢人時 | 80h/月 | 20h/月 | -75% |
| 異常反應 | 3h | 5min | -97% |
| 月報產出 | 3 工作天 | 即時 | -100% |
| 資料正確性 | 85% | 99% | +16% |
| 設備稼動率 | 65% | 75% | +15% |

### 方式 3：時間軸 / 動線

```
導入前
─┬─────┬─────┬─────┬─────┬──→ 時間
 異常   未察   未察   發現   處理
 0h    1h    2h    3h    3h+

導入後
─┬─────────────────────────────→ 時間
 異常→5min 告警→人員處理
```

## 必有元素

- [ ] **量化數字**：不能只寫「改善」，要寫多少
- [ ] **角色標示**：誰受益（廠長 / 業務 / 客服）
- [ ] **時間單位**：日 / 週 / 月 / 年
- [ ] **資料來源**：基線數字從哪量、目標從哪推
- [ ] **不確定標示**：假設、預期、待驗證的要標出

## 常見地雷

- ❌ 只寫「更快」、「更好」、「更省」 → 沒量化等於沒效益
- ❌ 改善數字看起來太美 → 客戶反而懷疑
- ❌ 沒寫基線是怎麼量的 → 上線後沒法驗
- ❌ 拿其他客戶的數字當這客戶的承諾 → 出事就攤
- ❌ 只看好處不看代價（學習曲線、流程改動）

## 進階：三維度比較

不只比現況，也比競品 / 替代方案：

| 維度 | 現況（不變） | 競品 A | 我們 |
|---|---|---|---|
| 巡檢人時 | 80h | 30h | 20h |
| 投資金額 | 0 | 600 萬 | 480 萬 |
| 導入期 | - | 8 個月 | 4 個月 |
| 上線後 6 個月 ROI | - | 待回收 | 已回本 |

## 反方審查問題

- 「省 75%」是怎麼算的？實際量測過嗎？
- 你的基線是用客戶內部資料還是行業平均？
- 達成這個效益的前提是什麼？前提沒達成怎麼算？
- 上線後實際達成率是百分之幾？前 N 個客戶實績多少？
