## 企業版加強章節

### A. 跨國 / 跨地部署（適用跨國集團）
- 部署地圖（哪些 region、為什麼）
- 跨境資料傳輸合法基礎（GDPR Art. 44-50、SCC、BCR）
- 資料主權矩陣（哪國資料能離境）
- 各國法規適用清單

### B. 集團 / 母公司治理
- 母公司核准流程
- 子公司部署順序
- 跨子公司資料共享規範
- 集團採購框架（Master Agreement）

### C. 完整資安治理
- ISO 27001 / SOC 2 Type II 認證（附證書）
- Pen Test 第三方報告（NDA 後提供）
- Threat Model（STRIDE 完整表）
- 應急響應 Runbook
- 資安事件 SLA（含 GDPR 72hr）
- IR 演練計畫
- BCP / DR 完整文件

### D. 合約附錄完整版
- MSA（Master Service Agreement）
- DPA（Data Processing Agreement）
- BAA（Business Associate Agreement，HIPAA 適用）
- NDA
- SLA 詳版（含違約金）
- Service Catalog
- Sub-processor 名單
- IP 處理條款
- 終止與退場條款
- 智財權與授權

### E. 採購合規（政府 / 大型企業特有）
- 政府採購法適用條款
- 押標金 / 履約保證金
- 評選方式（最有利標 / 最低標）
- 廠商盡職調查文件
- 反賄賂 / 利益衝突宣告
- 母公司財務健全證明

### F. 法規對應矩陣完整版
| 法規 / 標準 | 適用範圍 | 我方對應章節 | 證明文件 |

| ISO 27001 | 通用 | §X | 證書 |
| SOC 2 Type II | 雲服務 | §X | 報告 |
| GDPR | 歐盟用戶 | §X | DPA |
| HIPAA | 健康資料 | §X | BAA |
| PCI-DSS | 信用卡 | §X | AOC |
| ... | | | |

### G. ESG / 永續（上市櫃適用）
- 系統本身的碳足跡
- 對客戶 ESG 報告的貢獻
- 雲端供應商的永續承諾（AWS Sustainability）
- 廢棄硬體回收

### H. 完整 SBOM 與供應鏈
- SPDX / CycloneDX JSON 附檔
- License 完整分類
- 高風險元件處理計畫
- 元件 escrow（如要求）
- 供應商盡職調查

### I. 完整 RTM
- 跨多個系統的需求追溯
- Cross-team dependency mapping
- Master schedule（含外部供應商）

### J. 完整變革管理
- ADKAR + Kotter 8 Steps
- Champion network 計畫
- 跨部門 / 跨子公司 rollout
- 每階段 KPI 與調整

- 客戶引述（附授權）

- 認證證書

### M. 退出機制完整版

## 企業版額外注意

### 簽核時間估算

| 客戶類型 | 提案到簽約 |
|---|---|
| 中大型上市櫃 | 6-12 週 |
| 跨國集團（母公司核准）| 12-24 週 |
| 政府 / 國營 | 12-24 週（含招標流程）|
| 金融（金管會委外）| 16-24 週 |
| 醫療大型醫院 | 12-20 週 |

### 投標 vs 提案

如果是政府 / 公開招標：

- 必讀招標公告完整本
- 規格內標明的「不得 / 必須」絕對遵守
- 押標金 / 履約保證金準備
- 評選簡報另準備（不是直接讀提案）

### 預提防的反方審查

企業 / 政府案的**特別反方題**：

- 你母公司在哪？是台資 / 陸資 / 外資？（國產化 / 兩岸要求）
- 你的雲端供應商會被美國 / 中國制裁影響嗎？
- 你能配合 ESG 評比（DJSI / CDP / MSCI）填問卷嗎？
- 你能接受我們派 1 名駐點 PM 嗎？費用？
- 你的員工會出差到我們的廠 / 機關嗎？接受我們的資安訓練？
- 系統交付後源碼歸誰？要 escrow 嗎？
