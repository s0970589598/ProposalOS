## 雙向追溯驗證

寫完後檢查兩件事：

### 痛點 → 功能（向前追溯）

每個痛點都有至少一個功能對應？

| 痛點 | 對應功能 | 缺漏？ |
|---|---|---|
| P-01 | FR-05 | 否 |
| P-02 | FR-08 | 否 |
| P-03 | — | **是 → 補需求或讓客戶確認不做** |

### 功能 → 痛點（向後追溯）

每個功能都有對應的痛點？

| 功能 | 對應痛點 | 多做？ |
|---|---|---|
| FR-05 | P-01 | 否 |
| FR-08 | P-02 | 否 |
| FR-10 | — | **是 → 為什麼要做？砍掉或確認** |

## 狀態流程

```
規劃中 → 開發中 → 測試中 → UAT 中 → 已驗收 → 已上線 → 維運中
                                  ↓
                              退回 → 開發中
```

## 維護建議

| 階段 | 維護者 | 頻率 |
|---|---|---|
| 需求訪談 | PM | 每次訪談後更新 |
| 提案設計 | PM + Tech Lead | 每次設計變更 |
| 開發 | Dev Lead | 每週 |
| 測試 | QA | 每天 |
| 上線後 | Service Owner | 每月 |
