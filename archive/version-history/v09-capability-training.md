# V9｜能力訓練版

## 主題

「我自己怎麼變強」— 個人能力建構。

## 為什麼獨立出來

V1-V8 都在談「提案內容該寫什麼」。
V9 提出：流程再完整，靠的還是人。

→ 能力建構**不是提案內容**，要拉出來獨立成 playbook，不混進框架。

## V9 提到的核心能力

| 能力 | 練法 |
|---|---|
| 產業敏感度 | 每週讀產業報告 |
| 訪談技巧 | 結構化訪談、五次「為什麼」|
| 結構化思考 | 每案結構化覆盤 |
| 寫作能力 | 多寫、多改 |
| 簡報能力 | 錄影自我檢討 |
| 商務直覺 | 報價結構分析 |
| 法務常識 | 讀 DPA、SLA、SOW |
| 資安常識 | 跟 CISO 對話 |
| AI 使用力 | 找 AI 出錯 |

## V9 提到的方法

- 每案復盤
- 錯誤案例庫
- 提案審查 checklist
- 必讀書 / 必懂框架

## 對應到 V12

→ 完整保留進 [playbook/skill-building.md](../../playbook/skill-building.md)
→ 從框架本體獨立出去，跟 12 模組分開

## 為什麼這個分離很重要

| 不分開 | 分開（V12 做法）|
|---|---|
| 提案模板裡有「能力建構」章節 → 客戶看到很奇怪 | 提案模板乾淨 |
| 能力建構變成提案的責任 → 顧問會混淆 | 能力建構是個人事務 |
| 跟客戶交付物混在一起 → 內外不分 | 內外分清 |

## 保留的價值

- 能力建構作為長期工程
- 「框架不能取代人」的提醒
- 復盤、錯誤案例庫等具體方法
