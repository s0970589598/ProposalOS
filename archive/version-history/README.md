## V12 收斂公式

```
V12 = V7 骨幹（擴成 12 模組，吸收 V4 / V5 / V10 修正）
    + V11 落地補強（8 強制檢查項）
    + V2 / V5 產業議題（變成 industry-addons）
    + V3 AI 角色定位（變成 playbook/ai-assistance.md）
    + V9 能力建構（變成 playbook/skill-building.md，獨立出來不混進提案）
    + V1 視覺產出（變成 templates/ 與每模組「輸出物」清單）
    + V6 深度（保留為 archive 備查，不直接使用）
    + V8 工具評估（變成 research/tools-evaluation.md）
```

## 為什麼保留全部版本

- **不是因為捨不得**，是因為日後爭執時可以查證
- 過去做過的決策、被取代的概念都有紀錄
- 新人 onboarding 看演化軌跡可以快速理解 V12 為什麼長這樣
